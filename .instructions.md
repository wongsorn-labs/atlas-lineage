# Atlas Lineage AI Agent Instructions

Core instructions for all AI agents (Claude, Copilot, etc.) working with this repository.

## Conventional Commits (CRITICAL)

**Every commit message MUST follow this exact format:**
```
<type>(<scope>): <subject>
```

- **Type**: One of `feat`, `fix`, `test`, `refactor`, `docs`, `perf`, `chore`, `ci`, `style`
- **Scope**: One of `api`, `web`, `db`, `e2e`, `shared` (REQUIRED, in parentheses)
- **Subject**: Imperative, lowercase, no period, max 50 chars
- **No blank lines** between header and body

**Valid examples:**
- `feat(api): add person search endpoint`
- `fix(web): correct map marker positioning`
- `test(db): add encryption edge cases`
- `refactor(shared): simplify Zod schema validation`
- `chore(ci): update GitHub Actions workflow`

**Invalid examples (will be rejected):**
- `add new feature` ❌ (missing type and scope)
- `API: add search` ❌ (missing scope parentheses)
- `feat: add feature` ❌ (missing scope)
- `feat(api): Add Search Endpoint` ❌ (subject not lowercase)

When asked to suggest a commit message, commit code, or generate a changelog, **ALWAYS output in the format above**. Lefthook + commitlint will reject any other format.

## Project Structure

```
packages/shared   → TypeScript types + Zod schemas (source of truth for all)
packages/db       → Drizzle ORM queries + AES-256-GCM encryption
apps/api          → NestJS REST API (port 3001)
apps/web          → React 19 + Vite SPA (port 5173)
apps/e2e          → Playwright end-to-end tests
```

## Key Workflows

| Task | Command |
|------|---------|
| Install | `pnpm install` |
| Dev (API + web) | `pnpm dev` |
| Unit/integration tests | `pnpm test` |
| E2E tests | `pnpm test:e2e` |
| Database generate | `pnpm db:generate` |
| Database migrate | `pnpm db:migrate` |

## Rules Before Suggesting Changes

1. ✅ Types defined in `packages/shared` FIRST
2. ✅ API changes use NestJS module patterns (service → controller)
3. ✅ Database changes require `pnpm db:generate` migration
4. ✅ New packages export from `packages/shared` (single source of truth)
5. ✅ Field encryption: use `packages/db/src/crypto.ts` (never hardcode)
6. ✅ Tests added in `.spec.ts` files (Jest/Vitest)

## Common Tasks

### Adding a new API endpoint
1. Add type/DTO to `packages/shared`
2. Create service method in `apps/api`
3. Create controller endpoint in `apps/api`
4. Add test to `.spec.ts`
5. Commit: `feat(api): add <endpoint> endpoint`

### Database schema change
1. Edit schema in `packages/db/src/schema.ts`
2. Run `pnpm db:generate`
3. Review migration in `packages/db/drizzle/`
4. Run `pnpm db:migrate`
5. Commit: `feat(db): add <table/column>`

### Field encryption
- `birthPlace` and `notes` on `Person` are encrypted at rest
- Use `packages/db/src/crypto.ts` functions
- Null/empty strings stored as SQL NULL (not encrypted)
- `ENCRYPTION_KEY` env var required (64-char hex)
